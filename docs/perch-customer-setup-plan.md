# Perch Customer Setup Plan

Use this plan to onboard Perch in Life@Perch. Create these as tasks on the `Perch Customer Setup` board, then move each task through `New`, `To Do`, `Blocked`, `Hold`, and `Completed`.

## Task Cards

| Status | Task | Description | Acceptance Check |
| --- | --- | --- | --- |
| New | Confirm Perch project scope | Capture contacts, expected users, core workflows, reporting needs, and launch target date. | Scope, owner, and target date are recorded in the task notes. |
| New | Confirm existing services | Verify the current PocketBase and Ollama gateway connections are the intended services for Perch. | Login, task loading, and agent model loading use the existing configured endpoints. |
| New | Create Perch area boards | Confirm boards exist for `PerchGroup`, `ACI`, `Connect`, `TML`, and `Verify`. | Each area is available in the task board selector. |
| New | Confirm Microsoft 365 links | Replace placeholder area website links with the correct production URLs. | Outlook, Teams, and each area website opens from Quick Links. |
| New | Run end-to-end smoke test | Test login, task creation, assignment, Kanban movement, comments, refresh, and chat. | Smoke test notes are attached and no launch blockers remain. |
| New | Handover to Perch | Share the dashboard URL, support path, escalation contact, and usage notes. | Perch confirms access and understands the support route. |
